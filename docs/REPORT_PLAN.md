# Report Plan

## ICBC-Style Constraints

- IEEE 2-column conference format
- Full papers at ICBC 2025 were limited to 8 pages excluding references
- Relevant paper sections required by your course:
  - title
  - author information
  - abstract
  - introduction
  - related work
  - methodology
  - experimental results
  - conclusion
  - references

## Practical Target

Aim for:

- 6 to 8 pages of main content
- 1 to 1.5 pages of references

## Recommended Page Budget

### Page 1

- Title and author block
- Abstract
- Introduction

### Page 2

- Related Work
- System Model / Threat Model

### Page 3

- Methodology / Framework Design
- Architecture explanation

### Page 4

- Smart Contract Design
- Off-chain Simulation and Verification

### Page 5

- Experimental Setup
- Results table

### Page 6

- Results discussion
- Limitations
- Conclusion

### Pages 7-8 if needed

- Expanded discussion
- Future work
- More detailed figure/table support

## Writing Order

Write in this order, not paper order:

1. Methodology
2. Experimental Setup
3. Results
4. Introduction
5. Related Work
6. Conclusion
7. Abstract

This is the easiest way because:

- methodology comes directly from your code
- results come directly from your evaluation output
- abstract is easiest to write after everything else is known
